Informations to Start

With this plugin you can pricing the files and groups to selling it , and recive the payments to your paypal or Stripe account automaticly ( WebMasters Only )

You need only : PayPal Business Account or Stripe account, and you can create an REST API APP for paypal or get Stripe API keys # for developers : you can create a paypal app if your account is personal , but you will use the sandbox API (TEST) until you upgrade your account to buisness and for Stripe you will use the test api keys until you active your account

After you create your REST API APP you will got ( Client ID ) and ( Client Secret ) . set them in {olang.CONFIG_KLJ_MENUS_KLEEJA_PAYMENT} , and the ( Currency Code ) it have to be 3 Letter Only EX: USD , EUR , CHF check the accepted currencies code here
Dowload Link Expire After x Days ( 0 ) avilabe forever :
after the user make a apyment , he will recaive an e-mail with download link , and he can not download the bought file if he did not have an cookie in his browser , so this will help you to set the limit of days that he can access to the file.
note : if you give the access to user to access bought files page, the cookies time is not important , becuse the user will have unlimit accecc time .
Percentage of owner's earnings (10 -> 90) % :
this is the Percentage of the File owner , if a user upload a file and he want to sell it , and another person want to buy this file , he will recaive a selected Percentage from the file price ,
note : he will recaive the Percentage only if the group of the user have the acces to "Recaive Profits" , it's not important if the group don't have the permissions
Min/Max Price Limit :
when the group of the user have the acces to "Recaive Profits" permissions , he can set the price of his files ,
with this two option you can set the limit of the file price that the user can set it .
note : this options will be applied with user area only , not in the admin page

To pricing a file you neet to go to ( Pricing a File ) in the Side menu , with the file id or file link you can open file details , after clicking file details you will see some details about the file and at the end you will see a form to set the price , and if the file was paid , you can change the price or set the price to 0 (zero) to make the file for free again
and if you want to set a price for a group go to ( users & Groups -> Edit Data ) and you will find ( Join price ) .
Note : the default groups and the default registeration group can not be paid

File Owner and website founders can download the file for free ( Founder Role is not admins group , the founders can give founder Role to any member in kleeja even if he is not in admins group )

After you open kj_payments CP you can find some informations about payments ( Transactions numbers and amounts , and Net Profits )
all transactions : is the number of all approved payments when you started using this plugin ,
monthly transactions : is the number of all payments of this month with the net profits and it will be reseted evry month
daily transactions : is the number of all payments of today only and it will be reseted evryday
you can access to the old payments and amounts for any day or month by archives . (to access to monthly archive , make the day empty and select the month and year )
Only the website founders can access to kj_payments control Panel
Note : if you see that a method display the number of amount without writting ( Net Profit ) that mean the amount is writed with fees
The Files/Images URLs have to be from ( Row ID DataBase ) . Check Here

when you give "share_profit" permission to any group , they will have access to "Payments Control" Page , and also , they will have permission to pricing their files in selected limits .
and it will be able for them to receive the selected Percentage of the file price .
and they can ask to withdraw their balance to their paypal account .
if a user requist to withdraw an amount from hes account ;
you will see it in payouts request list , and you will see also 'check' button ,
if you canceled the payout , the amount will back to the user balance , and done
and if you accepted it , the balance will be sent to paypal user email , ( the email that the user registered it in kleeja )
after that the payout state will be 'SENT' , that mean we said to paypal to sent the amount to the user , but the payout is not confirmed yet
any payout with state 'SENT' , when you view it , you see in the footer to check the payout , is it received or not yet ,
note : most of the time the payout take 1 min to confirm it , if taked more then 1 min , check user email , is it regestered in paypal or not
if the email is not registered , the money will back to your account after 30 days .
